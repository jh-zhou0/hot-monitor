# 热点分析框架（Claude 执行）

本 Skill 不调用 OpenRouter 或任何外部 LLM API。采集脚本输出原始候选后，由 **当前对话中的 Claude** 按本文档完成分析。

## 分析维度

1. **真实性**：来源是否可靠？是否有事实依据？是否营销号/标题党/假冒？
2. **热度**：对 AI 领域影响程度？是否重大更新/发布/突破？
3. **时效性**：是否最新消息？若事件早于 30 天，应显著降低时效分。
4. **关键词相关性**：是否**直接**关于用户监控词？例如监控「Claude Sonnet 5」但内容是「Claude 3.5 Sonnet」，应判不相关，评分不超过 20。

## 评分标准（0–100）

| 分数 | 等级 | 说明 |
|------|------|------|
| 90–100 | 紧急 | 重大突破/发布（新模型、重大功能） |
| 70–89 | 重要 | 版本升级、重要合作 |
| 50–69 | 一般 | 行业动态、人事变动 |
| 30–49 | 偏低 | 评论、旧闻翻新 |
| 0–29 | 噪音 | 假消息/营销/与关键词不相关 |

- **≥80**：紧急热点，在简报中置顶标注
- **≥60**：值得纳入「重要」区块
- `is_genuine: false`：单独列入「已过滤」并说明理由

## 批处理建议

- 每批最多 **5** 条（与 Web 版 `analyzer.ts` 一致），避免上下文过长
- 对 `collect.py` 的 `results` 数组按 `index` 对齐输出

## 输出 JSON 结构

分析完成后先输出结构化 JSON（便于程序化处理），再按 `report-template.md` 写人类可读简报：

```json
{
  "keyword": "用户关键词",
  "analyzed_at": "ISO8601",
  "results": [
    {
      "index": 0,
      "title": "原标题",
      "source_type": "bing",
      "url": "https://...",
      "is_genuine": true,
      "score": 85,
      "summary": "一句话中文摘要",
      "analysis": "简短分析理由"
    }
  ],
  "stats": {
    "total": 12,
    "genuine": 8,
    "urgent": 2,
    "avg_score": 62.5
  }
}
```

## 用户 Prompt 模板（构建分析请求时）

```
用户监控的关键词是「{keyword}」。以下是搜索到的 {n} 条结果，请逐一分析，
特别注意每条是否真的与「{keyword}」直接相关：

[0] 标题: ...
来源: bing
内容: ...

（逐条列出 collect.py 输出中的 results）
```

只返回 JSON 时：仅输出上述 JSON，不要多余说明。需要简报时：JSON 之后另起一节 Markdown。
