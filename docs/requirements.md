# AI 热点监控系统 - 需求文档

## 项目背景

Kayson 作为 AI 编程博主，需要第一时间获取 AI 领域热点（如大模型更新），不能依赖人工手动搜索，而是利用工具自动发现指定的热点变化、发现最新热点，并及时发送通知。

## 核心功能

### F1: 关键词监控

- 用户手动输入要监控的关键词（如 "GPT-5"、"Claude"、"Cursor"）
- 系统从多个信息源搜索该关键词相关内容
- 利用 AI 识别内容真伪（过滤假冒、营销号、标题党）
- 当确认为真实热点时，第一时间发送通知

### F2: 热点自动发现

- 每隔一段时间（用户可配置间隔），自动搜集指定范围内的热点
- 范围由用户定义（如 "AI编程"、"大模型"）
- AI 对热点进行评分（0-100），高分热点优先展示和推送
- AI 生成中文摘要，方便快速了解

### F3: 热点展示

- 响应式 Web 页面，兼容桌面和移动端
- 赛博朋克/暗黑科技风视觉风格
- 支持按来源、评分、时间筛选
- 展示 AI 验证状态（真/假/待验证）

### F4: 通知推送

- 浏览器 Web Push 通知（关闭网页后也能收到）
- 邮件通知（可选）
- 可配置推送阈值（最低评分）
- 免打扰时段设置

### F5: Agent Skills（后续）

- 封装为 Agent Skills 技能
- 能够交给其他 AI 来监控和发现热点

## 数据源

| 来源 | 方式 | 说明 |
|------|------|------|
| 网页搜索 | 通过搜索引擎 URL 爬虫获取 | 不需要 API key，注意频率控制 |
| Twitter(X) | twitterapi.io API | 需要 API key，获取实时推文 |

要求：同时从多个信息源获取信息，避免单一信息源的局限性。

## 非功能需求

- 本地一键运行（`npm run dev`）
- 轻量敏捷，不过度工程化
- 频率控制，避免被封禁
- AI 识别假冒内容，减少噪音
