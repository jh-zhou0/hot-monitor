# AI 热点监控系统 - 需求文档

## 项目背景

Kayson 作为 AI 编程博主，需要第一时间获取 AI 领域热点（如大模型更新），不能依赖人工手动搜索，而是利用工具自动发现指定的热点变化、发现最新热点，并及时发送通知。

## 核心功能

### F1: 关键词监控

- 用户手动输入要监控的关键词（如 "GPT-5"、"Claude"、"Cursor"）
- 系统从 7 个信息源并发搜索该关键词相关内容
- 利用 AI 识别内容真伪（过滤假冒、营销号、标题党）
- 当确认为真实热点时，通过 WebSocket 第一时间推送到浏览器通知面板

### F2: 热点自动发现

- 每隔 60 分钟（可按关键词单独配置）自动采集一次
- 范围由用户定义的监控关键词决定
- AI 对热点进行评分（0-100），划分低/中/高三个等级
- AI 生成中文摘要，方便快速了解
- 高等级热点（≥80分）标记为紧急热点

### F3: 热点展示

- 响应式 Web 页面，兼容桌面和移动端
- 赛博朋克/暗黑科技风视觉风格（深色 + 霓虹色 + 粒子背景）
- 多维度筛选：
  - 维度一（卡片）：总热点 / 今日热点 / 紧急热点 / 监控词
  - 维度二（标签）：按来源筛选（Bing / DuckDuckGo / Google / 搜狗 / HN / 微博 / Twitter）
  - 维度三（搜索框）：关键词全文搜索
- 热点卡片展示：等级标签、来源标签、匹配度、AI 摘要、原文链接

### F4: 通知推送

- WebSocket 实时推送：采集到高分热点立即通知，页面打开时实时收到
- 浏览器通知面板：🔔 图标，未读红点计数，展示通知历史
- 邮件通知（可选）：配置 SMTP 后启用
- 可配置推送阈值（最低评分，默认 60 分）
- 免打扰时段设置

### F5: Agent Skills（待开发）

- 封装为 Agent Skills 技能
- 能够交给其他 AI 来监控和发现热点

## 数据源

| 来源 | 类型 | 需要 Key | 说明 |
|------|------|----------|------|
| Bing | 搜索引擎 | 否 | 国内可用，主力来源 |
| DuckDuckGo | 搜索引擎 | 否 | 国内可能超时，自动降级 |
| Google | 搜索引擎 | 否 | 需代理，自动降级 |
| 搜狗 | 搜索引擎 | 否 | 国内可用 |
| HackerNews | 技术社区 | 否 | 官方 Algolia API，免费 |
| 微博 | 社交媒体 | 否 | 热搜榜 + 关键词搜索 |
| Twitter/X | 社交媒体 | 是 | 需配置 twitterapi.io key |

所有来源并发采集，任意来源失败自动降级，不影响整体流程。

## 非功能需求

- 本地一键运行（`npm run dev`）
- 轻量敏捷，不过度工程化
- 频率控制，避免被封禁（搜索引擎 ≤ 2次/分钟）
- AI 识别假冒内容，减少噪音
- 数据持久化到本地 SQLite 文件
